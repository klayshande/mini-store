Vanilla JavaScript vs jQuery
Tarea Vanilla JavaScript jQuery
Seleccionar un elemento document.querySelector('#id') $('#id')
Escuchar un evento elemento.addEventListener('click', fn) $('#id').on('click', fn)
Solicitar datos a una API fetch(url).then(...) o await fetch(url) $.ajax({url, success: fn})

En este proyecto se utilizó Vanilla JavaScript porque el navegador moderno ya incluye fetch, querySelector y addEventListener de forma nativa, con soporte completo para async/await. Esto elimina la necesidad de cargar una librería adicional (jQuery) solo para tareas que el propio lenguaje ya resuelve de forma eficiente y con mejor rendimiento.

localStorage vs sessionStorage vs cookies
Característica localStorage sessionStorage Cookies
Duración de los datos Permanece hasta que se borre manualmente Se borra al cerrar la pestaña/navegador Configurable (puede expirar en minutos, días o nunca)
Capacidad aproximada ~5-10 MB ~5-10 MB ~4 KB
¿Se envía al servidor? No No Sí, en cada petición HTTP
Caso de uso típico Preferencias, datos que deben persistir entre sesiones Datos temporales de una sola sesión (ej. un formulario en progreso) Autenticación, identificadores de sesión que el servidor necesita leer

En esta práctica se usó localStorage para guardar los usuarios agregados manualmente, porque se necesitaba que esos datos persistieran incluso después de recargar la página, y no era necesario que el servidor los leyera (a diferencia de una cookie), ni que desaparecieran al cerrar la pestaña (a diferencia de sessionStorage).
